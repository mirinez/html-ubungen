Responsive Bilder: picture

Beispiel

Erklärung

  • Das "picture" Element ermöglicht mehrere Bildquellen für verschiedene Bildschirmgrößen.
  • Die "source" Elemente definieren Bedingungen für die Anzeige.
  • Das "img" Element dient als Fallback.
  • "bild-gross.jpg" wird geladen, wenn der Bildschirm mindestens 800px breit ist.
  • "bild-mittel.jpg" wird für Bildschirme mit mindestens 450px verwendet.
  • "bild-klein.jpg" dient als Fallback.

srcset

Das "srcset" Attribut erlaubt es, mehrere Bildquellen mit unterschiedlichen Auflösungen anzugeben.

Beispiel (1x / 2x)

In diesem Beispiel wird "bild-klein.jpg" für normale Displays verwendet, während "bild-gross.jpg" für hochauflösende Displays geladen wird.

Beispiel mit Breiten (w) + sizes

Das "sizes" Attribut gibt an, welche Bildgröße für verschiedene Bildschirmbreiten verwendet werden soll.

  • Der Browser wählt automatisch die passende Bildquelle basierend auf Bildschirmgröße und Auflösung.
  • Dies verbessert Ladezeit und Bildqualität.
  • Breiten-Deskriptoren (w) sind ideal für responsive Designs.
  • Auflösungs-Deskriptoren (x) sind ideal für feste Anzeigegrößen.

Lazy Loading

Lazy Loading lädt Bilder erst, wenn sie im sichtbaren Bereich erscheinen.

  • "loading="lazy"" aktiviert Lazy Loading.
  • Eager: Bild wird sofort geladen.
  • Ohne Attribut entscheidet der Browser automatisch.

Audio in HTML einbetten

Das HTML "audio" Element wird verwendet, um Audiodateien direkt in eine Webseite einzubinden.

Audio

  • "audio": definiert ein Audio-Element.
  • "controls": fügt Steuerelemente hinzu.
  • "controlsList="nodownload"": entfernt die Download-Option (nicht in allen Browsern gleich).
  • "source": definiert die Quelle und das Format.
